# Security Policy

## Reporting a Vulnerability

Please report security issues privately to the repository maintainer (for example via GitHub private security reporting or direct maintainer contact) before opening a public issue.

When reporting:
- Describe impact and reproduction steps clearly.
- Include affected commit/version if known.
- Share minimal logs needed to understand the issue.

## Safe disclosure guidelines

- Do **not** post real server IP addresses or secrets in public issues.
- Do **not** publish sensitive logs publicly.

## Supported Versions

Security fixes are provided for:
- `main` branch
- latest commit
